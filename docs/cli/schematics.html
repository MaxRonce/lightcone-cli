Execution Steps

1
Resolve config from ~/.prism/config.yaml; prompt for permission tier if missing.
2
Create project directory structure: scripts/, results/, universes/.
3
Write astra.yaml (boilerplate or migrated), universes/baseline.yaml, Containerfile, requirements.txt.
4
Write .prism/dagster.yaml (SQLite at results/.dagster/) and .prism/prism.yaml (target key).

Copy bundled plugin files from wheel into .claude/ — scripts, hooks, skills, agents, guides.

9
If --sub-analysis: wire new sub-analysis into parent astra.yaml and all universe files.

Execution Steps

1
Load astra.yaml, recursively resolve sub-analysis tree.
2
Determine target: --target flag → .prism/prism.yaml → ~/.prism/config.yaml → "local".
3
Unless --no-build: auto-build all container images (docker/podman/podman-hpc depending on target backend).
4
Create ASTRAContainerRunner with resolved backend + target config.
5
Build Dagster Definitions from build_definitions(): one asset per output, deps from recipe.inputs.
6
Filter to requested outputs (if any); resolve topological order.
7
Call dagster.materialize(assets, resources=...) — runner dispatches each asset to its backend.
8
Docker/local backend: inject universe decision params as --key value CLI args, run recipe command.
9
SLURM batch backend: generate sbatch script → sbatch → poll sacct until COMPLETED/FAILED.
10
On Docker failure: fall back to local execution with a warning.

Build (or migrate) all container images declared in astra.yaml.

prism build [--force] [--runtime docker|podman|podman-hpc] [--dry-run]
Reads / Inputs

→
External Programs
Docker/Podmandocker build -t <tag> -f <file> <ctx>local target
podman-hpcpodman-hpc build -t <tag> -f <file>HPC / SLURM target
podman-hpcpodman-hpc migrate <tag>push image to compute nodes
Inspectdocker image inspect <tag>check if already built (skip if exists & no --force)
→
Output
Image tagprism-<project>-<12-char-sha256>content-addressed: SHA256(Containerfile + dep files)
HPC storepodman-hpc image storemigrated to compute nodes

Execution Steps

1
Load astra.yaml, collect all unique container: values (analysis-level + per-recipe overrides).
2
Auto-detect runtime: SLURM target → podman-hpc; otherwise detect docker or podman locally.
3
For each container spec: if it's a file path → build; if it's an image name + SLURM → podman-hpc migrate.
4
Compute content-addressed tag: SHA256(Containerfile + DEPENDENCY_FILES) → prism-<name>-<hash[:12]>.
5
Skip build if image already exists (unless --force).

prism dev

Launch Dagster webserver UI for asset graph visualisation and run history.

prism dev [--port/-p PORT] [--universe/-u UNIVERSE]
Reads / Inputs
Specastra.yaml
Config.prism/dagster.yamlsets DAGSTER_HOME
→
Files Created (temp)
Temp/tmp/prism_defs_*.pyephemeral Dagster definitions file; deleted on exit
→
External Programs
Dagsterdagster-webserver-f <temp_defs.py> -h 0.0.0.0 -p <port>

Execution Steps

1
Load astra.yaml and determine universe (default: baseline).
2
Write ephemeral /tmp/prism_defs_*.py that calls build_definitions() with no_build=True.
3
Set DAGSTER_HOME to .prism/ (falls back to project root if .prism/dagster.yaml absent).
4
Launch dagster-webserver -f <file> -h 0.0.0.0 -p <port>; block until user Ctrl-C.
5
On exit (normal or error): delete temp file.

Target Config Schema (written to ~/.prism/targets/)

·
backend: slurm | docker | local | venv
·
connection: hostname, username (for remote SLURM)
·
account: HPC allocation ID (site-specific suffix added automatically, e.g. _g for GPU on Perlmutter)
·
container_runtime: docker | podman | podman-hpc | singularity
·
scheduler: site key, container_flags, extra_slurm_args
·
max_nodes / max_walltime_minutes / max_concurrent_jobs

Loop Mechanics

1
Check state: if .claude/ralph-loop.local.md exists, offer resume or fresh start.
2
Plan: create .prism/plans/build-plan-<universe>.md; user approves before loop starts.
3
Activate: setup-prism-build.sh --activate writes ralph-loop state file; Stop hook is now armed.
4
Each iteration: Survey → Decide → Write code → prism run → git commit.
5
Stop hook re-injection: PostToolUse Stop hook reads ralph-loop state and re-injects prompt for next iteration automatically.
6
Exit: emit <promise>BUILD_COMPLETE</promise>; Stop hook deactivates loop; state file removed.
